# 팩트체크 프로세스

OpenClaw 콘텐츠 생성 시 **필수** 팩트체크 절차입니다.

---

## 🎯 목표

**사실 정확도 90% 이상** 달성

---

## 📋 팩트체크 워크플로우

```
1. 콘텐츠 생성 (LLM)
   ↓
2. 초안 작성 완료
   ↓
3. 🔍 팩트체크 단계 (필수)
   ↓
   3-1. 주요 클레임 식별
   3-2. 웹 검색으로 검증
   3-3. 공식 출처 확인
   3-4. 허위/오류 정보 수정
   ↓
4. 검증 완료
   ↓
5. 승인 → 발행
```

---

## ✅ 필수 팩트체크 항목

### 장소/관광지 포스트

#### 1. 기본 정보
- [ ] **주소**: 정확한 도로명 주소
- [ ] **운영시간**: 요일별, 계절별 시간
- [ ] **휴무일**: 정기 휴무, 공휴일
- [ ] **연락처**: 전화번호, 공식 웹사이트

#### 2. 시설 정보
- [ ] **건물 구조**: 층수, 주요 공간
- [ ] **편의시설**: 주차장, 화장실, 엘리베이터 등
- [ ] **특수 시설**: 옥상, 정원, 카페 등 (실제 존재 여부)

#### 3. 가격 정보
- [ ] **입장료**: 정확한 금액, 할인 정보
- [ ] **주차비**: 요금, 무료 시간
- [ ] **무료 입장**: 조건, 시간대

#### 4. 교통 정보
- [ ] **대중교통**: 지하철역, 버스 노선
- [ ] **도보 거리**: 인근 역에서 실제 소요 시간

---

### 전시/이벤트 포스트

#### 1. 전시 정보
- [ ] **전시명**: 정확한 제목 (오타 없이)
- [ ] **기간**: 시작일, 종료일
- [ ] **장소**: 전시 공간 (관명, 층수)

#### 2. 내용 검증
- [ ] **작가/작품**: 실존 여부, 참여 작가
- [ ] **전시 주제**: 공식 설명과 일치
- [ ] **관람 정보**: 촬영 가능 여부, 특별 관람 조건

---

### 맛집/카페 포스트

#### 1. 영업 정보
- [ ] **영업시간**: 요일별 시간, 브레이크 타임
- [ ] **휴무일**: 정기 휴무, 연휴
- [ ] **예약**: 예약 가능 여부, 방법

#### 2. 메뉴/가격
- [ ] **대표 메뉴**: 실제 판매 여부
- [ ] **가격**: 정확한 금액 (인상/인하 반영)
- [ ] **특별 정보**: 런치 세트, 할인 등

## 🔍 팩트체크 방법

### 1. 공식 출처 우선 (1차 출처)
```
공식 웹사이트 > 공공기관 정보 > 뉴스 > 블로그
```

**예시**:
- 국립현대미술관 → mmca.go.kr (공식)
- 문화재청 관리 → heritage.go.kr
- 관광공사 → visitkorea.or.kr

### 2. 웹 검색 활용

```bash
# 장소 기본 정보
"[장소명] 운영시간 휴무일 2026"

# 전시 정보
"[미술관명] 2026 전시 일정"

# 최신 정보 확인
"[장소명] site:공식도메인"
```

### 3. 교차 검증
- 최소 2개 이상의 독립적 출처 확인
- 정보 불일치 시 공식 출처 우선

### 4. 날짜 확인
- **치명적**: 과거 정보를 현재/미래로 작성
- 정보의 최신성 확인 (뉴스 날짜, 업데이트 날짜)

---

## 🚨 치명적 오류 (즉시 수정)

### Red Flag 항목
1. **존재하지 않는 시설** (예: 없는 옥상정원, 층수 오류)
2. **허구의 전시/이벤트** (예: 실제 없는 전시명)
3. **잘못된 운영 정보** (예: 휴무일 오류로 방문 불가)
4. **틀린 가격** (예: 2배 이상 차이)
5. **폐업한 장소** (최신 영업 여부 확인)

## 📊 팩트체크 기준

### 정확도 점수
```
90-100%: ✅ 발행 가능
70-89%:  ⚠️ 수정 후 재검토
0-69%:   ❌ 전면 재작성
```

### 검증 레벨
- **Verified**: 공식 출처로 확인됨
- **Likely**: 신뢰할 만한 출처로 확인
- **Unknown**: 출처 불명확
- **False**: 명백한 오류

## 📝 수정 프로세스

### 1. 오류 발견 시
```bash
# 1. 이슈 생성
git checkout -b fix/fact-check-mmca

# 2. 수정
# 파일 편집

# 3. 재검증
npx tsx scripts/fact-check.mts [수정한 파일]

# 4. 커밋
git add [수정한 파일]
git commit -m "fix: 국립현대미술관 포스트 팩트체크 수정

- 5층 → 3층 (지하1층 + 지상3층)
- 옥상정원 삭제
- 2026년 실제 전시로 교체
- 입장료 정정 (통합권 10,000원)

정확도: 10% → 95%"

# 5. Moltbook 재공유 (필요시)
```

### 2. 발행 후 오류 발견 시
1. 즉시 draft: true로 변경
2. 수정 완료 후 재발행
3. Moltbook 공지 게시 (선택)

## 🔄 프로세스 통합

### 콘텐츠 생성 프롬프트에 반영

```typescript
// src/generator/prompts.ts

export const FACT_CHECK_REQUIREMENTS = `
CRITICAL: 사실 정확도 90% 이상 필수

팩트체크 필수 항목:
1. 주소, 운영시간, 휴무일 - 공식 웹사이트 확인
2. 건물 층수, 시설 - 실제 존재 여부 확인
3. 전시/이벤트 - 2026년 실제 일정 확인
4. 가격 - 최신 요금 확인

허위 정보 절대 금지:
- 존재하지 않는 시설/공간 언급 금지
- 확인되지 않은 전시/이벤트 작성 금지
- 불확실한 정보는 "공식 웹사이트 참조" 안내
`;
```

### 일일 자동화 워크플로우

```bash
npm run daily:run
   ↓
초안 생성 (drafts/)
   ↓
🔍 npm run fact-check:all  ← 신규 추가
   ↓
팩트체크 통과 → 승인
   ↓
npm run publish
```

## ⚠️ 주의사항

### 하지 말아야 할 것
- ❌ LLM 생성 정보를 그대로 신뢰
- ❌ 개인 블로그만 참조
- ❌ 과거 정보를 현재로 작성
- ❌ "아마도", "대략" 같은 불확실한 표현

- ✅ 공식 웹사이트 최우선 확인
- ✅ 정보 출처 명시
- ✅ 불확실하면 작성하지 않음
- ✅ 정기적 정보 업데이트
